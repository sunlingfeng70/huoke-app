# Obsidian 获客文档整理归类 Skill

## 适用场景

vault 中已有通过搜索保存的小红书笔记/评论（`小红书评论/` 目录），需要：

1. 自动扫描所有笔记和评论，提取用户数据
2. 按等级、标签、地域、时间线多维度归类
3. 生成索引页，方便销售和运营快速查找

## 前置条件

### 依赖

本项目使用 `uv` 管理依赖：

```bash
uv add openpyxl
```

### 输入

Obsidian 仓库中已有 `小红书评论/` 目录，包含通过 Streamlit 三步工作流保存的笔记和评论。

## 工具位置

```
xhs_obsidian_organizer/xhs_obsidian_organizer.py
```

## 操作步骤

### Step 1: 运行整理

```bash
uv run python xhs_obsidian_organizer/xhs_obsidian_organizer.py <Obsidian仓库路径>

# 示例
uv run python xhs_obsidian_organizer/xhs_obsidian_organizer.py ./vault
uv run python xhs_obsidian_organizer/xhs_obsidian_organizer.py ~/MyVault --verbose
```

### Step 2: 在 Obsidian 中浏览

打开 Obsidian 仓库，从 **「📊获客总览」** 开始浏览。

## AI 调用

在 Streamlit AI 对话中，LLM 可直接调用 `organize_notes` 工具一键完成整理归类。

## 生成的索引页

运行后在 `小红书获客/索引/` 目录生成以下页面：

| 文件 | 内容 | 用途 |
|---|---|---|
| `📊获客总览.md` | 数据概览、热门标签、地域分布、笔记列表 | 全局入口 |
| `🟢A类高意向客户.md` | A类客户按评分降序列表 + 建议开场白 | 销售优先联系 |
| `🟡B类中意向客户.md` | B类客户列表 + 建议开场白 | 培育跟进 |
| `🔵C类低意向客户.md` | C类客户列表 | 内容培育池 |
| `🏷️按标签归类.md` | 按"学习/变现/就业/创作/创业/地域"标签分组 | 按需求维度查找 |
| `📍按地域归类.md` | 按IP属地分组 | 按城市合伙人区域查找 |
| `📅时间线.md` | 按日期排列，每日A类高亮 | 追踪获客趋势 |
| `🎯销售行动总表.md` | A+B类合并，按优先级排序，含建议开场 | 销售工作台 |

## Obsidian 仓库结构

```
MyVault/
├── 小红书评论/              ← 已有（来源数据）
│   ├── 2026-07-09_agent学习/
│   └── 今日评论汇总/
└── 小红书获客/             ← 自动生成
    └── 索引/
        ├── 📊获客总览.md
        ├── 🟢A类高意向客户.md
        ├── 🟡B类中意向客户.md
        ├── 🔵C类低意向客户.md
        ├── 🏷️按标签归类.md
        ├── 📍按地域归类.md
        ├── 📅时间线.md
        └── 🎯销售行动总表.md
```

## 核心特性

- **自动解析**：从笔记和评论中提取用户数据
- **多维归类**：等级、标签、地域、时间四个维度
- **Obsidian 双链**：索引页通过 `[[]]` 链接回原始笔记
- **评分排序**：所有列表按评分降序，优先级一目了然
- **销售行动总表**：A+B类合并，直接可作为销售每日工作清单
- **AI 可调用**：注册为 LLM function calling 工具，对话中直达

## 常见问题

**Q: 每次运行为什么覆盖旧的索引？**
A: 索引是实时生成的视图，每次运行重新扫描全部数据并重建。原始笔记不会被修改。

**Q: 支持多天的数据吗？**
A: 支持。会自动扫描 `小红书评论/` 下所有 `.md` 文件，合并后统一归类。

**Q: 数据格式变了会怎样？**
A: 脚本依赖笔记和评论的标题结构解析，如果保存格式变化可能需要调整解析逻辑。
